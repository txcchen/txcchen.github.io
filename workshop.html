About This Project

This project is a web application developed with the MERN tech stack that allows users to manage various projects that involve multiple participants and streamlines the check-in process via a project dashboard and custom submission form, titled [Workshop].

I have completed a prototype of the application deployed with Render.com

Designer & Developer

Workshop is intended for artists to more easily manage art projects as the project organizer or moderator.

Inspiration & Problem Definition

I have participated in Secret Santa Art Exchanges as an artist and also as a moderator. For these art exchanges, participating artists are required to submit progress checks on pre-determined deadline dates to ensure that they are working on their art piece. Moderators are required to check and make sure that each participant has submitted a progress check at those times.

Previously, I was one of the moderators for my friend's Secret Santa Art Exchange. We kept track of the participants and progress checks with Google Spreadsheets and had participants submit their work-in-progress (WIP) via a Google Form. We would then go through each form and check it off on the spreadsheet. At the time, I thought It would be convenient if the form directly linked to the spreadsheet so that the artist's WIP is automatically linked. This would make it easier to manage the art exchange and also minimize human-error of possibly checking off the wrong person.

So this year, I decided to create a web application that will do just that - manage art projects.

Keeping in tune with the holiday spirit, the application is named Workshop inspired by Santa's Workshop. Workshop can manage not only art exchanges (Secret Santas, Valentines Exchanges, etc), it can also be used for managing art zine projects or any project that requires participants to submit WIPs and progress checks.

Home page for not logged in users. Will be updated to contain more details and infographics.

Features

Features of Workshop include:

  • Projects : moderator can create, edit, delete projects - all projects are shown on the dashboard
  • Roles : moderator can add and delete roles (e.g "artist", "writer", etc)
  • Timeline : moderator can add deadlines/tasks
  • Participants : moderator can add and delete participants and assign them a role

Managing all parts of a project is done via the Dashboard or in the project page.

Submission Form

The submission form is for the moderator to share with the participants for the participants to fill out. Once submitted, the participant's progress will be updated in the project page.

User Login

I decided to use Google authentication to allow users to log in with their google accounts. This way, I only store the user's Google email and profile picture.

Database

I used MongoDB to store data. I have 3 models - User, Projects, and Participants. As shown in the gif below, when the participant submits their progress via the form, the information will be updated in the participant document. This is also how the Project is updated with tasks and roles.

Reflection & Future

I learned how to develop a web application using MERN and managed to implement useful features. Next, I will be improving the application, with to-dos including redesigning the UI, restructuring the database so that Roles can be linked to the Tasks belonging to that role, using ContextAPI to access user information instead of prop drilling and add features such as adding collaborators/other moderators and making the project page publicly viewable/sharable for projects that don't require the art pieces to be private before the deadline. My goal is to have Workshop be usable to manage art projects.
